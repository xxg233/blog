# 配置java环境（windows 10） + 踩坑

# 前言

今天在github上看到一个很好的开源博客项目，名字叫halo github地址 (opens new window)，官网 (opens new window)，社区 (opens new window)。由于运行该项目需要配置java环境，为了记录下踩坑的经过，也为了方便以后自己再次安装，所以有了下文。

# 安装java jdk

我选择的是jdk版本为1.8.0_281，下载与安装jdk的步骤如下

  1. 首先进入java官网 下载地址 (opens new window), 选择如下版本，点击下载

  1. 如果已经登录账号了就可以直接下载，如果没有登录账号就会跳到登录页面，如果没有账号的话可以自己注册一个，也可以像我一下，百度搜索 Oracle 账户共享 (opens new window)，直接使用好心人的账号进行登录，登录完之后就可以正常下载啦

  1. 直接双击下载好的jdk安装包，按照默认配置一步一步往下安装直到完成就好啦

# 安装好jdk之后，我们需要手动配置环境变量（这里最容易出错）

先按正常步骤来吧（需要注意踩坑的地方后面再讲）

  1. 在电脑桌面 右键点击 “此电脑”的“属性”选项

  1. 选择“高级系统设置”选项

  1. 点击下面的“环境变量”选项

  1. 点击“系统变量”下面的”新建“选项(需要是系统变量，如果点错了变成用户变量了可能会出问题)

  1. 在”变量名“ 处填上 ”JAVA_HOME“ ”变量值“为JDK安装路径，我的路径是”C:\Program Files\Java\jdk1.8.0_281“ 点击”确定“选项

  1. 在”系统变量“中找到”Path“

  2. 选中”Path“点击”编辑“选项

  1. 直接点击新增 分别填入C:\Program Files\Java\jdk1.8.0_281\bin 以及 C:\Program Files\Java\jdk1.8.0_281\jre\bin （注意：其中C:\Program Files\Java\jdk1.8.0_281\ 有的博客说可以用之前配置的%Java_Home%代替，但是问题就在于我的 JAVA_HOME 是在用户变量中，而不是系统变量，也就是说path这个变量中的 %JAVA_HOME% 对于javac来说可能没生效！所以我建议各位直接用绝对路径写）

  1. 除了上面这种新增的方法之外我们还可以点击编辑文本，将引号里面的全部复制“%Java_Home%\bin;%Java_Home%\jre\bin;”，到“变量值”栏的最前面，“确定”

  1. 有人说要额外配置CLASSPATH 变量，但是我发现好像现在不需要配置这个了，所以这一步可以省略。配置步骤：在“系统变量”栏，“新建”，“变量名”为“CLASSPATH”，“变量值”为“.;%JAVA_HOME%\bin;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar”，“确定”，最后点击“环境变量”最下面的“确定”选项（尤其要注意前面有个小数点、分号和百分号 ./%）

  1. 回到电脑桌面，按快捷键“Win+R”，输入“cmd”， 分别输入java + enter，javac + enter，java -version + enter，其中enter代表输入完按键盘回车， 如果出现下面这几张图，就证明配置成功啦

  1. 如果输入javac + enter显示（javac不是内部或外部命令，也不是可运行的程序或批处理文件），但是java 和 java -version正常的话，可以试试以下两种解决方法
  • 第一个坑，请把%JAVA_HOME%替换成绝对路径（替换完之后记得重新打开一个cmd运行命令）
  • 第二个坑，当你的 PATH 这个是直接复制过来的，对于win10来说，不能直接粘贴进来
